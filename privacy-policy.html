1. Scope

WarpDownload is a macOS download manager. It can receive downloads that you add manually, import in batches, send from supported browser integrations such as the optional WarpDownload Chrome extension, route through the optional local browser bridge, index completed files for on-device Spotlight search, or share selected metadata across your Macs through optional iCloud sync features. This Privacy Policy covers the information involved in those features.

When you use a supported WarpDownload browser integration, the browser integration can send supported download requests to the local WarpDownload app running on the same device through http://127.0.0.1. The browser handoff flow is intended for local-device communication and not for routing those requests through WarpDownload-operated remote servers.

This policy does not govern the privacy practices of websites, file hosts, CDNs, cloud providers, or Apple's services. When you download from a third-party source, that source may receive requests from your device according to its own privacy practices.

2. Data We Handle

2.1 Data you enter or choose

  • Download URLs, file names, tags, notes, checksum values, workspace selections, and scheduling details that you enter in WarpDownload.
  • Save locations and folders that you choose, including tag-specific folders, workspace folders, or network storage locations you authorize.
  • Optional request metadata you provide, such as User-Agent, Referer, Cookie, usernames, or passwords.

2.2 Data processed to run downloads

  • Download progress, status, connection counts, speed samples, timestamps, retry state, resume state, and destination information.
  • Response metadata such as content type, redirect destinations, file size information, range support, and server-provided file-name suggestions.
  • Checksums, receipt details, and history records for completed or failed downloads.

2.3 Saved credentials

If you choose to save site credentials for a host, WarpDownload stores them in the macOS Keychain so they can be reused for later downloads from that host.

Saved site credentials use macOS Keychain

2.4 Clipboard monitoring

Clipboard monitoring is optional. If you enable it, WarpDownload checks the clipboard for supported direct download links so it can prefill the add-download flow. Clipboard monitoring is intended to detect direct file links, not to read unrelated clipboard contents for analytics or advertising.

Optional feature

2.5 Browser bridge and browser-assisted capture

If you enable the local browser bridge and use a supported WarpDownload browser integration such as the Chrome extension, WarpDownload can receive a locally forwarded payload that may include the URL, file name, MIME type, page title, save path, referrer, cookies, and request headers supplied by the browser integration.

Sensitive browser request data, including cookies and authorization headers, is disabled by default. It is only included if you explicitly enable that behavior in Settings.

The browser integration may also read the active tab URL or a supported browser download URL so it can prefill or hand off the download you asked WarpDownload to manage. WarpDownload uses that browser-derived data only for the requested download handoff and related download management features.

2.6 iCloud queue sync

If you enable iCloud queue sync, WarpDownload shares pending download jobs across your Macs using Apple's iCloud key-value storage. Shared items can include the URL, optional file name, save path, selected headers, request metadata, timestamps, and the device name that added the job.

Cookies, authorization headers, and referrers are excluded from iCloud sync by default. They are only included if you explicitly opt in.

2.7 iCloud history and preference sync

If you enable iCloud history sync, WarpDownload can share selected download history metadata across your Macs using Apple's CloudKit private database. Synced history metadata can include the file name, source URL, status, size, dates, tags, workspace label, checksum details, and the device name associated with the record.

If you enable iCloud preference sync, WarpDownload can sync a limited set of app preferences through Apple's iCloud key-value storage, such as theme mode, auto-open-completed preference, notification toggles, and the global speed limit setting.

Downloaded files themselves are not synced, and local file paths are not shared across your Macs as part of history sync.

2.8 Spotlight indexing

WarpDownload can index completed downloads into the macOS Spotlight index on the same Mac so you can find them through system search. Indexed metadata can include the file name, file type, source host name, tags, workspace name, file size, and completion date.

WarpDownload does not place the full original download URL, signed query strings, or secret tokens into Spotlight metadata.

2.9 Apple-managed purchase and entitlement data

WarpDownload uses Apple's StoreKit and App Store systems to determine whether a trial, subscription, or lifetime unlock is active. Payment processing, billing information, subscription renewal, and purchase restoration are handled by Apple, not by WarpDownload directly.

3. How We Use Data

  • To start, schedule, accelerate, pause, resume, retry, verify, and complete downloads that you request.
  • To organize files using your chosen folders, tags, workspaces, naming rules, and post-download actions.
  • To maintain download history, receipts, and recovery information so interrupted downloads can continue safely.
  • To support optional features you enable, such as browser-assisted capture, clipboard suggestions, iCloud sync, Spotlight indexing, Finder tags, notifications, Music import, and saved credentials.
  • To determine access to paid features through Apple's StoreKit APIs.

Based on the current app implementation, WarpDownload does not include third-party advertising SDKs, analytics SDKs, or cross-app/cross-site tracking libraries.

Browser-derived data is used only to hand off supported downloads to the local WarpDownload app, preserve authenticated download context when you explicitly opt in, and show browser bridge status. WarpDownload does not sell that data and does not use it for advertising, profiling, or unrelated analytics.

4. Permissions and Device Access

  • Notifications: WarpDownload requests notification permission only when needed for download alerts or when you test notifications.
  • Folder access: WarpDownload uses standard macOS folder pickers when you choose a save location, default folder, or tag folder. It stores security-scoped bookmarks so it can continue writing to those folders later.
  • iCloud: iCloud queue sync, history sync, and preference sync remain off unless you enable them.
  • Browser bridge: The local browser bridge remains off unless you enable it.
  • Browser integration permissions: Supported WarpDownload browser integrations may use browser permissions to detect supported downloads, read the current tab URL for manual handoff, store extension settings locally, read site cookies when required for authenticated downloads, and communicate with the local WarpDownload app over localhost.
WarpDownload is designed so sensitive features are turned on by you, not silently enabled in the background.

5. When Data Leaves Your Device

Most WarpDownload data stays on your Mac. Data leaves your device only in the following situations:

  • Browser handoff to the local app: If you use a supported WarpDownload browser integration, the selected download request data is sent to the local WarpDownload app on the same device through localhost so the app can start the download.
  • Download requests: When you download a file, your Mac contacts the file host or source server and may send the request metadata required to complete that request.
  • iCloud queue sync: If you enable iCloud queue sync, pending download jobs are shared through Apple's iCloud infrastructure.
  • iCloud history and preferences sync: If you enable these features, selected history metadata and a limited set of app preferences are shared through Apple's iCloud infrastructure.
  • App Store purchases: Subscriptions, trials, restores, and entitlement validation are handled through Apple services.
  • Support communication: If you contact support, the information you choose to send will be used to respond to your request.
Important: If you enable sensitive browser capture or opt in to syncing sensitive request data through iCloud, cookies, authorization headers, and related session context may be processed so authenticated downloads can work correctly.
WarpDownload does not use the browser handoff flow to transmit your selected download requests to WarpDownload-operated remote servers for advertising, profiling, or unrelated analytics.

6. Security and Storage

  • Saved site credentials are stored in the macOS Keychain.
  • Download records, tags, preferences, receipts, diagnostics logs, and recovery state are stored locally on your Mac.
  • Downloaded files are stored in the save locations you choose or authorize.
  • The local browser bridge listens only on localhost when enabled and is designed for local-device communication.
  • Spotlight indexing is local to the Mac where indexing is enabled and uses the system Spotlight index on that device.
  • Supported WarpDownload browser integrations use bundled extension code and local communication with the app. They are not designed to download or execute remote code as part of the browser-to-app handoff flow.

No method of storage or transmission is completely secure. You are responsible for choosing trusted download sources and protecting access to your Mac, Apple account, and saved credentials.

7. Your Choices and Controls

  • You can turn clipboard monitoring on or off in Settings.
  • You can enable or disable the local browser bridge in Settings.
  • You can keep sensitive browser headers and iCloud request context disabled, or opt in only when needed.
  • You can enable or disable iCloud queue sync, iCloud history sync, and iCloud preference sync in Settings.
  • You can install, disable, or remove a supported WarpDownload browser integration through your browser's extension controls.
  • You can remove saved credentials from the app's credential manager.
  • You can clear iCloud-shared queue items from within the app.
  • You can delete downloads, history items, tags, workspaces, and local files according to the controls WarpDownload provides.
  • You can manage or cancel subscriptions through your App Store account settings.

8. Children's Privacy

WarpDownload is not directed to children. If you believe information from a child has been provided through the app or related support channels inappropriately, contact us so we can review and address the issue.

9. Changes to This Policy
